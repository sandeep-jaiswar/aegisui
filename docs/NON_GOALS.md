# AegisUI — Non-Goals

The following are **explicitly out of scope** and will not be added.

* HTML compatibility
* CSS or cascading style rules
* JavaScript or scripting support
* Runtime mutation of UI trees
* Animations or transitions
* Asynchronous effects
* Implicit layout or styling
* String-based identifiers
* WYSIWYG tooling
* Accessibility abstractions (handled elsewhere)

If a feature requires any of the above, it does **not** belong in AegisUI.

> **AegisUI values predictability over flexibility.**
